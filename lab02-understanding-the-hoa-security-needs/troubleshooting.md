# 🛠️ Troubleshooting Guide — Lab 02: Understanding the HOA Security Needs

---

## Issue 1: `sudo snap install threat-dragon` fails or Snap not working

### ❌ Problem
Snap commands fail or the snap service is not running.

### ✅ Cause
- `snapd` not installed
- `snapd` service not enabled/started

### ✅ Resolution
```bash
sudo apt update
sudo apt install -y snapd
sudo systemctl enable --now snapd
````

Validate:

```bash
snap version
snap list
```

---

## Issue 2: `openvas` package not available (Ubuntu)

### ❌ Problem

```text
E: Package 'openvas' has no installation candidate
```

### ✅ Cause

On newer Ubuntu versions, OpenVAS package name is replaced by **GVM** packages.

### ✅ Resolution

Install GVM instead:

```bash
sudo apt install -y gvm
```

## Issue 3: Feed sync is slow or fails

### ❌ Problem

`greenbone-feed-sync` takes too long or errors.

### ✅ Causes

* Network restrictions in cloud lab
* Feed servers slow/unreachable
* DNS issues

### ✅ Resolution

Retry and ensure connectivity:

```bash
ping -c 3 archive.ubuntu.com
nslookup feed.community.greenbone.net
sudo greenbone-feed-sync
```

## Issue 4: `gvm-setup` completes but services not reachable

### ❌ Problem

Setup finishes but web UI/services do not respond.

### ✅ Causes

* Services not started correctly
* Firewall rules blocking ports
* Not enough system resources

### ✅ Resolution

Check service status:

```bash
sudo gvm-check-setup
sudo systemctl status redis-server
sudo systemctl status postgresql
```

Re-run setup validation:

```bash
sudo gvm-check-setup
```

---

## Issue 5: Credentials generated by `gvm-setup` are sensitive

### ⚠️ Problem

Setup prints an admin password on screen.

### ✅ Security Handling

* Do **not** commit real credentials into GitHub.
* Replace with `REDACTED` in repo if needed.
* Store securely (password manager) if reused.

Recommended practice:

* keep secrets out of repository
* treat lab credentials as ephemeral
